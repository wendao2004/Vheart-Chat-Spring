
# 本规范为基础的规范记录，后续持更新，开发前请阅读
#1.编码规范
## 1）基础规范
* 文件名，方法名，以及定义的class，用英文或者拼音（建议用英文），不要用拼写简写。
* 所有文件名约定下使用短横线连接（不用驼峰方式）
* 方法名使用驼峰方式
* 方法和css 实现要简洁，同样的功能或同样样式尝试共用
## 2）uni组件及第三方组件使用规范
* 使用的组件最好能支持android，ios，及微信，快手，抖音小程序，百度小程序

#通常云函数后端采用分层设计
## 1) Api-Auth   统一认证中心	（多 APP 登录、注册、密码加密、账号校验）				
* 核心入口，所有 APP、所有云函数都调用它做身份验证
	
## 2)Api-Token  登录态校验	（Token 验证、刷新 Token、判断是否登录、过期处理）
* 所有业务云函数第一步都调用它校验用户是否登录
 
## 3)Api-User   用户中心		（获取用户信息、修改昵称 / 头像、个人资料、用户状态）
* 依赖 Api-Auth，每个 APP 独立用户数据
	
## 4)Api-Common 全局配置	    （APP 版本号、公告、功能开关、全局参数）
* 前端启动时调用，拉取配置
	
## 5)Api-Config 通用工具	    （短信发送、验证码、数据加密、通用接口）
* 零散小功能汇总，避免建太多小云函数
	
## 6)Api-Upload 文件上传	    （头像上传、图片上传、文件上传（uniCloud 云存储））
* 所有 APP、用户头像 / 聊天图片都用它

#Vheart-Chat/
* ├── src/
* │   ├── bean/             # 1. 实体层（TS类型，前后端数据结构统一）
* │   │   ├── UserBean.ts   # 用户实体
* │   │   ├── ChatBean.ts   # 消息实体
* │   │   └── AuthBean.ts   # 登录/注册参数
* │   ├── net/              # 2. 网络层（对接后端3个云函数）
* │   │   ├── ApiAuth.ts    # 对接后端：Api-Auth（登录/注册）
* │   │   ├── ApiApp.ts     # 对接后端：Api-App（所有业务）
* │   │   ├── ApiUpload.ts  # 对接后端：Api-Upload（上传）
* │   │   └── HttpRequest.ts # 统一封装云函数请求
* │   ├── data/             # 3. 数据层（本地存储/全局状态）
* │   │   ├── UserData.ts   # 用户信息缓存
* │   │   └── AppData.ts    # 全局配置缓存
* │   └── model/            # 4. 业务层（页面直接调用）
* │       ├── AuthModel.ts  # 登录注册业务
* │       ├── UserModel.ts  # 用户资料业务
* │       └── ChatModel.ts  # 聊天业务
* └── pages/                # 页面层（只调用model，不写逻辑）

#Vheart-Chat/uniCloud-aliyun/cloudfunctions/

* ├── Api-Vheart-Chat       # 主业务：用户/好友/聊天/配置（所有功能合并）
